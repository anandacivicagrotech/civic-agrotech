# 🟥 Slide 0 – Problem & Motivation

### 🎯 วัตถุประสงค์
อธิบายปัญหาหลักของระบบแจ้งเตือนปัจจุบันและเหตุผลที่ต้องพัฒนา "RD Team Alert & Notification System"

---

### ❌ ปัญหาที่พบ (Pain Points)

**ระบบแจ้งเตือนปัจจุบัน:**
- 😴 **ทีม RD เพิกเฉย** - ไม่เห็นว่าเร่งด่วนแค่ไหน
- 🤔 **แจ้งเตือนไม่มีบริบท** - เช่น "EC = 2.3" แต่ไม่บอกว่าควรทำอะไร
- 📊 **ข้อมูลเป็นตัวเลข** - ไม่ใช่ภาษาที่นักวิจัยเข้าใจ
- ⏰ **ไม่รู้ว่าเร่งด่วนหรือไม่** - ทุกอย่างดูเหมือน "แจ้งเตือน" เท่ากัน
- 🔔 **Alert Fatigue** - แจ้งบ่อยเกินไป จนเพิกเฉยทั้งหมด

**ตัวอย่างแจ้งเตือนปัจจุบันที่ไม่ดี:**
```
⚠️ Alert: EC = 2.3 (Zone A)
```
→ **RD คิดว่า:** "แล้วไง? ควรทำอะไร? เร่งด่วนไหม?"

---

### 💸 ผลกระทบที่เกิดขึ้น
- **Ignored Alerts**: ทีม RD เพิกเฉย 60-70% ของ alerts
- **Late Response**: ตอบสนองช้า ปัญหาลุกลามแล้ว
- **Experiment Failure**: การทดลองเสียหายเพราะไม่ได้แก้ไขทันเวลา
- **Lost Productivity**: เสียเวลาในการวิเคราะห์ว่าควรทำอะไร

**ตัวอย่างผลกระทบ:**
```
Before: แจ้ง "EC = 2.3" → ทีม RD เพิกเฉย
        → ใบไหม้ → ทดลองเสียหาย → สูญเสีย 10-15% yield
```

---

### ✅ แนวทางแก้ไข
- พัฒนา **RD Team Alert & Notification System** แบบมีบริบท
- **Contextual Alerts** - บอกว่าเกิดอะไร, ทำไม, และต้องทำอะไร
- **Impact-Based Prioritization** - แยกความเร่งด่วน (P0, P1, P2)
- **Actionable Recommendations** - บอกขั้นตอนแก้ไขชัดเจน
- **RD-Specific Language** - ใช้ภาษาที่นักวิจัยเข้าใจ

**ตัวอย่างแจ้งเตือนใหม่ที่ดีกว่า:**
```
🟡 WARNING - EC สูงเกินไปในการทดลอง Oak Lettuce
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📍 Batch #2025-015 (Oak Lettuce, สูตร A)
📊 EC: 2.3 → เป้าหมาย: 1.6-1.8 (สูงกว่า +28%)

⚠️ ผลกระทบ: ใบอาจไหม้ (tip burn) ภายใน 24-48 ชม

✅ แนะนำ:
1. ลด EC ลงเป็น 1.6-1.8 ภายใน 2 ชม
2. เช็คว่าใส่ปุ๋ยเกินหรือเปล่า
```
→ **RD เข้าใจทันที** และรู้ว่าต้องทำอะไร!

---

### 💬 สรุปในประโยคเดียว
> จากแจ้งเตือนที่เพิกเฉย → แจ้งเตือนที่ตอบสนองทันที
> Civic AgroTech ต้องการ "ทีม RD ตอบสนองเร็วขึ้น 80%+" ด้วย Contextual Alerts

# 🟩 Slide 2 – System Overview

### 🎯 วัตถุประสงค์
แสดงภาพรวมของระบบและช่องทางการแจ้งเตือน

---

### ⚙️ โครงสร้างระบบ

```

[Sensor API] → [Context Builder] → [Alert Engine] → [Notification Channels]
     ↓              ↓                  ↓                    ↓
[Environmental] [Digital Logbook] [Priority Rules]  [LINE/Email/Dashboard]
[Data]         [Plant Health]     [Recommendations]

```

---

### 📦 ส่วนประกอบหลักของระบบ

| หมวด | ส่วนประกอบ | หน้าที่ |
|-------|---------------------------|----------|
| 📡 **Data Sources** | Sensor API + Digital Logbook + Plant Health | รวบรวมข้อมูลจากหลายแหล่ง |
| 🧠 **Context Builder** | Rule-based system | ดึงข้อมูล batch/experiment จาก Logbook |
| ⚡ **Alert Engine** | Threshold checking + Priority scoring | ตรวจสอบและประเมินความเร่งด่วน |
| 💡 **Recommendation Engine** | Rule-based + ML (future) | สร้างคำแนะนำการแก้ไข |
| 📱 **LINE Official Account** | LINE Messaging API | ส่ง alert แบบ rich message |
| 📧 **Email System** | SMTP / SendGrid | ส่ง non-critical alerts |
| 💻 **Web Dashboard** | React + API | แสดง alert history + acknowledge/resolve |
| 📲 **Mobile Push** | Firebase (optional) | แจ้งเตือนผ่าน mobile app |

---

### 🧠 System Capabilities

**Context Building:**
- 🔗 **Experiment-Aware**: รู้ว่าแต่ละ batch เป็นการทดลองอะไร
- 🌱 **Plant-Specific**: รู้ว่าพืชอะไร (Oak Lettuce, Butterhead)
- 📋 **Recipe-Aware**: รู้ว่าใช้สูตรอะไร (สูตร A, B, C)
- 📅 **Timeline-Aware**: รู้ว่าอยู่วันที่เท่าไร (Week 1, Week 2)

**Priority Scoring:**
- 🔴 **P0 - CRITICAL**: ต้องแก้ภายใน 30 นาที (PPFD = 0, Temp > 30°C)
- 🟡 **P1 - WARNING**: ต้องแก้ภายใน 2 ชม (EC/pH นอกช่วง 20%+)
- 🟢 **P2 - INFO**: ตรวจสอบภายใน 1 วัน (minor fluctuations)

**Recommendation Engine:**
- ✅ สร้างคำแนะนำเฉพาะเจาะจง (ลด EC จาก 2.3 → 1.7)
- ✅ บอก expected outcome (ถ้าไม่แก้ → ผลผลิตลด 10-15%)
- ✅ มี checklist สำหรับติดตาม

---

### 📸 การทำงานของระบบ

```
1. 📡 Sensor ตรวจจับค่าผิดปกติ (เช่น EC = 2.3)
   ↓
2. 🧠 Context Builder ดึงข้อมูล:
   - Batch #2025-015 (Oak Lettuce, สูตร A)
   - วันที่ 7/30 (ช่วงโต)
   - เป้าหมาย EC: 1.6-1.8
   ↓
3. ⚡ Alert Engine ประเมิน:
   - Priority: P1 (WARNING)
   - Impact: ใบอาจไหม้ภายใน 24-48 ชม
   ↓
4. 💡 Recommendation Engine สร้างคำแนะนำ:
   - ลด EC ลงเป็น 1.6-1.8 ภายใน 2 ชม
   - เช็คว่าใส่ปุ๋ยเกินหรือเปล่า
   ↓
5. 📱 ส่ง alert ผ่าน LINE + Dashboard
```

---

### 💬 สรุปในประโยคเดียว
> ระบบรวบรวมข้อมูลจากหลายแหล่ง สร้างบริบท และส่ง actionable alerts
> เพื่อให้ทีม RD ตอบสนองได้เร็วและแม่นยำ 🚨✨

### 🚨 Alert Priority Levels

**🔴 P0 - CRITICAL (ต้องแก้ภายใน 30 นาที)**
```
ตัวอย่าง:
- PPFD = 0 (ไม่มีแสง → พืชหยุดโต)
- Temp > 30°C (พืชตายได้)
- Pump failure (พืชขาดน้ำ)

การแจ้งเตือน:
- LINE + Push Notification (ทันที)
- Call/SMS (ถ้าไม่ acknowledge ภายใน 5 นาที)
```

**🟡 P1 - WARNING (ต้องแก้ภายใน 2 ชม)**
```
ตัวอย่าง:
- EC สูง/ต่ำกว่า target 20%+
- pH นอกช่วง
- CO₂ ต่ำกว่า target

การแจ้งเตือน:
- LINE + Dashboard
- Email (สำหรับ documentation)
```

**🟢 P2 - INFO (ตรวจสอบภายใน 1 วัน)**
```
ตัวอย่าง:
- Minor fluctuations (EC เพิ่ม 5%)
- ต้องเติมน้ำเร็วๆ นี้
- ถึงวันเก็บเกี่ยว (reminder)

การแจ้งเตือน:
- Dashboard only
- Email summary (รายวัน)
```

# 🟩 Slide 4 – System Operation Flow

### 🎯 วัตถุประสงค์
อธิบายลำดับขั้นตอนการทำงานของระบบและตัวอย่างการใช้งานจริง

---

### 🔄 ลำดับขั้นตอนการทำงาน

```

📡 Detect   →   🧠 Build Context   →   ⚡ Score Priority   →   💡 Generate Rec   →   📱 Notify

```

| ขั้นตอน | รายละเอียด | Processing Time |
|----------|-------------|-----------------|
| 📡 **Detect** | ตรวจจับค่าผิดปกติ<br>• Threshold checking<br>• Anomaly detection<br>• Compare with targets | Real-time |
| 🧠 **Build Context** | สร้างบริบท<br>• ดึงข้อมูล Batch/Experiment<br>• ดึงข้อมูล Plant type/Recipe<br>• ดึงข้อมูล Timeline | <2 วินาที |
| ⚡ **Score Priority** | ประเมินความเร่งด่วน<br>• Impact analysis<br>• Classify P0/P1/P2<br>• Determine notification channel | <1 วินาที |
| 💡 **Generate Rec** | สร้างคำแนะนำ<br>• Actionable steps<br>• Expected outcome<br>• Checklist | <2 วินาที |
| 📱 **Notify** | ส่งแจ้งเตือน<br>• LINE + Dashboard<br>• Email (ถ้าจำเป็น)<br>• Wait for acknowledgement | <1 วินาที |

---

### 📅 Implementation Timeline (2 เดือน)

**เดือนที่ 1: Alert Logic & Context**
- Week 1-2: ออกแบบ alert rules + context builder
- Week 3-4: พัฒนา recommendation engine

**เดือนที่ 2: Notification & Testing**
- Week 5-6: ตั้งค่า LINE Official Account + Dashboard
- Week 7-8: User testing กับทีม RD, ปรับ threshold, Presentation

---

### 💡 ตัวอย่างการใช้งานจริง

**Scenario: EC สูงเกินไปในการทดลอง Oak Lettuce**

**Step 1: Detect (ระบบตรวจจับ)**
```
📡 Sensor detects:
- Zone A, Batch #2025-015
- EC = 2.3 (high)
- Timestamp: 2025-11-15 10:30 AM
```

**Step 2: Build Context (สร้างบริบท)**
```
🧠 Context Builder queries Digital Logbook:
┌─────────────────────────────────┐
│ Batch #2025-015                 │
│ Plant: Oak Lettuce              │
│ Recipe: สูตร A                  │
│ Target EC: 1.6-1.8              │
│ Day: 7/30 (ช่วงโต)              │
│ Responsible: @นักวิจัย RD       │
└─────────────────────────────────┘

Historical data:
- Batch #2025-010 (Oak Lettuce, สูตร A):
  EC 2.2 → tip burn → yield ลง 12%
```

**Step 3: Score Priority (ประเมินความเร่งด่วน)**
```
⚡ Alert Engine evaluates:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Priority: 🟡 P1 - WARNING
Reason:
- EC 28% สูงกว่า target
- Oak Lettuce ไวต่อ high EC
- อยู่ในช่วงโต (day 7) - ช่วงสำคัญ

Expected Impact:
- ใบอาจไหม้ (tip burn) ภายใน 24-48 ชม
- Yield อาจลง 10-15%
- การทดลองอาจไม่ได้ผลตามที่คาด

Time to Fix: 2 ชม
```

**Step 4: Generate Recommendations (สร้างคำแนะนำ)**
```
💡 Recommendation Engine generates:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ แนะนำ:
1. ลด EC จาก 2.3 → 1.6-1.8 ภายใน 2 ชม
   → เติมน้ำประมาณ 30 ลิตร
2. เช็คว่าใส่ปุ๋ยเกินหรือเปล่า
3. ตรวจสอบใบอีกครั้งใน 12 ชม
4. ถ้าใบไหม้ → ถ่ายรูปส่งให้ทีม

📋 Checklist:
[ ] ลด EC ลงเป็น 1.7
[ ] เติมน้ำแล้ว
[ ] ตรวจสอบใบแล้ว (12 ชม)
[ ] แก้ไขเสร็จแล้ว
```

**Step 5: Notify (ส่งแจ้งเตือน)**
```
📱 LINE Message sent to ทีม RD:
┌─────────────────────────────────────────┐
│ 🟡 WARNING - EC สูงเกินไปในการทดลอง    │
│    Oak Lettuce                          │
│─────────────────────────────────────────│
│ 📍 Batch #2025-015 (Oak Lettuce, สูตร A)│
│ 📊 EC: 2.3 → เป้าหมาย: 1.6-1.8         │
│    (สูงกว่า +28%)                       │
│ ⏰ สถานะ: วันที่ 7/30 (ช่วงโต)          │
│                                         │
│ ⚠️ ผลกระทบ:                            │
│ - ใบอาจไหม้ (tip burn) 24-48 ชม        │
│ - ผลผลิตลด 10-15%                      │
│                                         │
│ ✅ แนะนำ:                               │
│ 1. ลด EC → 1.6-1.8 ภายใน 2 ชม         │
│ 2. เช็คว่าใส่ปุ๋ยเกินหรือเปล่า         │
│ 3. ตรวจสอบใบใน 12 ชม                   │
│                                         │
│ [✓ รับทราบ] [🔧 ดำเนินการแล้ว]         │
│ [📊 ดูรายละเอียด]                       │
└─────────────────────────────────────────┘

💻 Dashboard also updated with alert card
```

**Step 6: User Response (ผู้ใช้ตอบสนอง)**
```
นักวิจัย RD คลิก [🔧 ดำเนินการแล้ว]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ระบบถาม:
✅ คุณทำอะไรไปบ้าง?
[ ] ลด EC ลงเป็น 1.7
[ ] เติมน้ำแล้ว
[ ] ตรวจสอบใบแล้ว

นักวิจัยเลือก:
[✓] ลด EC ลงเป็น 1.7
[✓] เติมน้ำแล้ว (35 ลิตร)
[ ] ตรวจสอบใบแล้ว (จะเช็คใน 12 ชม)

ระบบบันทึก:
- Response time: 15 นาที
- Action taken: ลด EC, เติมน้ำ
- Status: Pending verification (12 ชม)
```

**Step 7: Follow-up (ติดตามผล)**
```
📱 12 ชม ต่อมา - ระบบส่ง reminder:
┌─────────────────────────────────────────┐
│ 🔔 REMINDER                             │
│ ตรวจสอบใบ Batch #2025-015              │
│─────────────────────────────────────────│
│ คุณแก้ไข EC เมื่อ 12 ชม ที่แล้ว        │
│ ได้เวลาตรวจสอบว่าใบดีขึ้นหรือยัง       │
│                                         │
│ [✓ ใบดีขึ้นแล้ว] [⚠️ ยังมีปัญหา]        │
└─────────────────────────────────────────┘

นักวิจัยตอบ: [✓ ใบดีขึ้นแล้ว]

ระบบบันทึก:
- Alert resolved successfully
- Time to resolution: 12 ชม
- Outcome: Positive
```

ผลลัพธ์:
→ ใบไหม้ → ทดลองเสียหาย → yield ลง 12%

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### 👥 ผู้ใช้งานระบบ (Target Users)

**ผู้ใช้งานหลัก (Primary Users):**
- 🌱 **ทีม RD / นักวิจัยพืช**
  - ได้รับ alert ที่มีบริบทชัดเจน
  - รู้ว่าต้องทำอะไร เร่งด่วนแค่ไหน
  - แก้ไขก่อนการทดลองเสียหาย

**ผู้ใช้งานรอง (Secondary Users):**
- 👨‍💼 **R&D Manager**
  - ดูว่าทีมตอบสนอง alert ไหม
  - วิเคราะห์ปัญหาที่เกิดบ่อย
- 💼 **Management**
  - ดู experiment health score

**💰 Value Proposition:**
> "แจ้งเตือน**แบบมีบริบท** บอกว่า **ต้องทำอะไร** และ **เร่งด่วนแค่ไหน**"
> → **📢 Effective Communication** (ทีม RD ตอบสนองเร็วขึ้น 80%+)

---

### 🎯 Success Metrics

**Response Metrics:**
- Response rate **>80%** (ตอบสนองภายใน 2 ชม)
- Alert accuracy **>90%** (ไม่ใช่ false alarm)
- Action completion **>85%** (ทำตาม checklist)
- User satisfaction **>4.5/5**

**Business Impact:**
- ลด Experiment Failure **50%**
- เพิ่ม Experiment Success Rate **40%**
- ลด Crop Loss **30%**
- ประหยัดต้นทุน **15-20%**

### 💡 Expected Outcomes

**Phase 1 (2 เดือน):**
- ✅ LINE Bot ทำงานได้ แจ้งเตือนแบบมีบริบท

- ✅ Checklist system ติดตามว่าแก้ไขแล้วหรือยัง
