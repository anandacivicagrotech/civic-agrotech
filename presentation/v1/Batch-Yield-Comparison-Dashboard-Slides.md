# 🟥 Slide 0 – Problem & Motivation

### 🎯 วัตถุประสงค์

---

### ❌ ปัญหาที่พบ
- **การเปรียบเทียบแบบ Manual**: ใช้ Excel หรือกระดาษ ทำให้เปรียบเทียบยาก
- **ข้อมูลกระจัดกระจาย**: ข้อมูลแต่ละ batch อยู่คนละที่ ค้นหายาก
- **ไม่เห็น Pattern**: ไม่รู้ว่าสูตรไหนได้ผลดี หรือตัวแปรไหนส่งผล
- **การเรียนรู้ช้า**: ต้องใช้เวลานานในการหา best practice
- **ขาดการวิเคราะห์เชิงลึก**: ไม่มี statistical analysis หรือ correlation

---

### 💸 ผลกระทบที่เกิดขึ้น
- **R&D Inefficiency**: ทำการทดลองซ้ำๆ โดยไม่เรียนรู้จากอดีต
- **Inconsistent Results**: ผลผลิตแปรปรวนสูง ไม่สม่ำเสมอ
- **Missed Opportunities**: พลาดโอกาสปรับปรุงที่อาจเพิ่มผลผลิต 10-15%
- **Time Waste**: ใช้เวลานานในการวิเคราะห์ข้อมูลแบบ manual

---

### ✅ แนวทางแก้ไข
- พัฒนา **Batch Yield Comparison Dashboard** เพื่อวิเคราะห์อัตโนมัติ
- **เปรียบเทียบผลผลิต** และ performance ระหว่าง batch
- **หา Best Practice** อย่างเป็นระบบด้วย data analytics
- **Visual Charts** ที่เห็นภาพชัดเจน และเข้าใจง่าย
- **เร่งการเรียนรู้ 3x เร็วขึ้น** เทียบกับการทดลองแบบเดิม

---

### 💬 สรุปในประโยคเดียว
> จาก Manual Experiments → Analytics-Based R&D  
> Civic AgroTech ต้องการ "เรียนรู้จากทุก batch ที่ปลูก"

### 📊 ข้อมูลที่วิเคราะห์ (Analysis Metrics)

**Primary Metrics:**
- 🌱 **Yield (kg)**: ผลผลิตรวม
- 📏 **Yield per Plant**: ผลผลิตต่อต้น
- 📅 **Growth Days**: จำนวนวันที่ปลูก
- 📊 **Success Rate**: เปอร์เซ็นต์พืชที่รอดตาย

**Recipe Variables:**
- ⚡ **EC Level**: ระดับธาตุอาหาร
- 🧪 **pH Level**: ความเป็นกรด-ด่าง
- 💡 **PPFD**: ความเข้มแสง
- 🌡️ **Temperature**: อุณหภูมิเฉลี่ย
- 🫧 **CO₂ Level**: ระดับคาร์บอนไดออกไซด์

**Quality Metrics:**
- 🥇 **Grade A %**: เปอร์เซ็นต์คุณภาพ A
- 📏 **Average Leaf Size**: ขนาดใบเฉลี่ย
- 🎨 **Color Uniformity**: ความสม่ำเสมอของสี
- 🌿 **Freshness Score**: คะแนนความสด

- **Analysis**: คำนวณ statistics และ correlations

---

### 🔄 ลำดับขั้นตอนการทำงาน
```

📊 Collect   →   🔄 Process   →   📈 Analyze   →   📋 Compare   →   💡 Recommend

```

| ขั้นตอน | รายละเอียด | Processing Time |
|----------|-------------|-----------------|
| 📊 **Collect** | ดึงข้อมูลจาก Digital Logbook<br>• Batch records<br>• Recipe parameters<br>• Yield data | Real-time |
| 🔄 **Process** | ทำความสะอาดข้อมูล<br>• Data validation<br>• Standardization<br>• Grouping | <5 นาที |
| 📈 **Analyze** | Statistical analysis<br>• Calculate averages<br>• Correlation analysis<br>• Trend detection | <2 นาที |
| 📋 **Compare** | เปรียบเทียบ performance<br>• Batch vs batch<br>• Recipe vs recipe<br>• Time series analysis | <1 นาที |
| 💡 **Recommend** | สร้าง insights<br>• Best practices<br>• Optimization suggestions<br>• Action plans | <30 วินาที |
