### **1. PFAL Efficiency Analyzer** 💡
**ระบบวิเคราะห์ประสิทธิภาพการปลูกและพลังงาน**

**สถานะ:** ✅ **APPROVED** - ผ่านการอนุมัติ

#### 🎯 **วัตถุประสงค์หลัก**
แก้ปัญหาต้นทุนพลังงานสูง (40-60% ของต้นทุนรวม) โดยการวิเคราะห์ประสิทธิภาพแบบครบวงจร

#### 🔧 **ฟีเจอร์หลัก**
- **Energy Monitoring**: ติดตามการใช้ไฟฟ้าแบบ real-time
- **Water Usage Tracking**: วัดการใช้น้ำในแต่ละระบบ
- **CO₂ Consumption Analysis**: ติดตามการใช้ CO₂
- **Yield Measurement**: บันทึกผลผลิตที่ได้
- **KPI Calculation**: คำนวณ kWh/kg, L/kg, g CO₂/kg
- **Cost Analysis**: วิเคราะห์ต้นทุนต่อหน่วยผลผลิต

#### 📊 **เป้าหมายที่วัดได้**
- ลดต้นทุนพลังงาน **15-25%**
- เพิ่มประสิทธิภาพการใช้น้ำ **10-20%**
- ปรับปรุง yield per square meter **5-15%**
- ROI ชัดเจนภายใน **6 เดือน**

#### 👥 **Target Users**
- 💼 Management (ดูต้นทุน, ROI)
- 👨‍💼 Operations Manager (ติดตาม efficiency)
- 🌱 นักวิจัย R&D (วิเคราะห์สูตรปลูก)

**Value:** **💰 Business Value** - ลดต้นทุน เพิ่มกำไร

### **2. Plant Health Monitoring System** 🌱
**ระบบตรวจสอบสุขภาพพืชด้วย Computer Vision**

**สถานะ:** ✅ **APPROVED** - ผ่านการอนุมัติ

#### 🎯 **วัตถุประสงค์หลัก**
ตรวจสอบคุณภาพและลักษณะของพืช เพื่อประเมินก่อนเกิดปัญหาร้ายแรง และแนะนำการปรับแก้ที่เหมาะสม

#### 🔧 **ฟีเจอร์หลัก**
- **Leaf Size Measurement**: วัดขนาดใบอัตโนมัติด้วย Computer Vision
- **Leaf Color Analysis**: วิเคราะห์สีใบ (เขียวปกติ, เหลือง, น้ำตาล, ไหม้)
- **Growth Tracking**: ติดตามการเจริญเติบโตในแต่ละช่วงเวลา
- **Anomaly Detection**: ตรวจจุดด่าง, เน่า, โรคพืช
- **Correlation Analysis**: เชื่อมโยงกับ environmental data (EC, pH, Temp, PPFD)
- **Actionable Recommendations**: แนะนำการแก้ไขทันที

#### 📊 **เป้าหมายที่วัดได้**
- **ตรวจพบปัญหาเร็วขึ้น 3-5 วัน** (ก่อนเห็นด้วยตาเปล่า)
- **ลดการสูญเสียผลผลิต 10-15%** (แก้ไขทันก่อนแพร่กระจาย)
- **เพิ่มความแม่นยำในการวินิจฉัย 90%+**
- **ลดเวลาตรวจสอบ 70%** (จาก manual inspection)

#### 👥 **Target Users**
- 🌱 นักวิจัยพืช / Agronomist (วิเคราะห์ปัญหา)
- 👨‍🌾 Crop Manager (ตัดสินใจแก้ไข)
- 👷 Operators (ได้รับ alert)

**Value:** **🔬 Proactive Prevention** - ป้องกันการสูญเสีย

### **3. Digital Logbook & Batch Tracker** 📋
**ระบบบันทึกข้อมูลและติดตามรอบการปลูกแบบดิจิทัล**

**สถานะ:** ✅ **APPROVED** - ผ่านการอนุมัติ

#### 🎯 **วัตถุประสงค์หลัก**
แทนที่การบันทึกแบบ manual (กระดาษ/Excel) ด้วยระบบดิจิทัลที่มี traceability เต็มรูปแบบ

#### 🔧 **ฟีเจอร์หลัก**

- **Digital Forms**: บันทึกข้อมูลผ่านมือถือ/คอมพิวเตอร์
  - วันเพาะ / วันย้าย / วันเก็บเกี่ยว
  - สูตรการปลูก (EC, pH, PPFD, Temp, CO₂)
  - ปริมาณเมล็ด, จำนวนต้น, ผลผลิต (kg)
  - ปัญหาที่พบ และการแก้ไข
- **Timeline View**: ดู timeline ของแต่ละ batch
- **Batch Comparison**: เปรียบเทียบ performance ระหว่าง batch

- **Export Reports**: ออกรายงาน PDF/Excel

- **เพิ่ม data quality** (ไม่มีข้อมูลสูญหาย/ผิดพลาด)
- **ลดเวลาค้นหาข้อมูล 80%** (จากค้นในกระดาษ/Excel)

#### 👥 **Target Users**
- 👷 Operators (บันทึกข้อมูลประจำวัน)
- 👨‍💼 Operations Manager (ดู progress ทุก batch)
- 🌱 นักวิจัยพืช (ย้อนดูข้อมูล experiment)

### **4. OCR-Based Harvest Data Entry Tool** 📸✍️
**เครื่องมือ OCR ช่วยบันทึกข้อมูลผลผลิตอัตโนมัติ**

**สถานะ:** 🆕 **NEW** - ปรับจาก Batch Yield Comparison Dashboard

#### 🎯 **วัตถุประสงค์หลัก**
ช่วยนักพัฒนาพืชบันทึกผลผลิตจากกระดาษ/บอร์ดให้เร็วขึ้นด้วย OCR และ Computer Vision

#### 💡 **ปัญหาที่พบ (Pain Point)**
นักพัฒนาพืชต้องบันทึกข้อมูลผลผลิตจากกระดาษหรือบอร์ดที่เขียนไว้ในโรงเรือน:
- ✍️ เขียนบนกระดาษ/บอร์ดขณะเก็บเกี่ยว
- 📋 กลับมาพิมพ์ใหม่ในคอมพิวเตอร์ (เสียเวลา 15-30 นาที/batch)
- ❌ ข้อมูลหาย หรือพิมพ์ผิด (5-10% ของข้อมูล)
- 😓 งานซ้ำซ้อน น่าเบื่อ

#### 🔧 **ฟีเจอร์หลัก**

**1. OCR Text Recognition**
- ถ่ายรูปกระดาษ/บอร์ดด้วยมือถือ
- OCR อ่านตัวเลข/ข้อความอัตโนมัติ
- รองรับ:
  - Batch ID
  - วันที่
  - น้ำหนัก (kg)
  - จำนวนต้น
  - ตัวเลขต่างๆ

**2. Smart Field Mapping**
- AI จับคู่ข้อมูลกับ field ที่ถูกต้องอัตโนมัติ
- เช่น "Batch #2025-001" → ใส่ใน Batch ID
- เช่น "125 kg" → ใส่ใน Yield field

**3. Manual Correction**
- แสดงผลที่อ่านได้ให้ user ตรวจสอบ
- แก้ไขได้ทันทีก่อน save
- เรียนรู้จากการแก้ไข (improve accuracy)

**4. Template Support**
- สร้าง template สำหรับแบบฟอร์มที่ใช้บ่อย
- ระบุว่า field ไหนอยู่ตรงไหนในภาพ
- เพิ่มความแม่นยำ 20-30%

**5. Direct Integration**
- บันทึกเข้า Digital Logbook (หัวข้อ 3) โดยตรง
- ไม่ต้องพิมพ์ใหม่

#### 📱 **การทำงาน**
```
📸 ถ่ายรูปกระดาษ/บอร์ด
   ↓
🔍 OCR อ่านข้อความ + ตัวเลข
   ↓
🤖 AI จับคู่ข้อมูลกับ field
   ↓
✏️ User ตรวจสอบ/แก้ไข (ถ้าจำเป็น)
   ↓
💾 บันทึกเข้า Digital Logbook อัตโนมัติ
```

#### 📊 **ตัวอย่างการใช้งาน**

**Before OCR (ปัจจุบัน):**
```
1. เก็บเกี่ยว → เขียนบนกระดาษ (5 นาที)
2. กลับออฟฟิศ → พิมพ์ใหม่ใน Excel (15 นาที)
3. Double-check (5 นาที)
---
รวม: 25 นาที/batch
```

**After OCR (ระบบใหม่):**
```
1. เก็บเกี่ยว → เขียนบนกระดาษ (5 นาที)
2. ถ่ายรูป → OCR อ่าน → ตรวจสอบ (3 นาที)
---
รวม: 8 นาที/batch (ประหยัด 17 นาที/batch)
```

#### 📊 **เป้าหมายที่วัดได้**
- **ลดเวลาบันทึกข้อมูล 65%** (จาก 25 นาที → 8 นาที)
- **ลด data entry error 80%** (จากการพิมพ์ผิด)
- **เพิ่มความสะดวก** (ทำที่ไหนก็ได้ผ่านมือถือ)
- **ประหยัดเวลา 3-5 ชม/สัปดาห์** สำหรับทีม RD

**OCR Engine:**

### **5. RD Team Alert & Notification System** 🚨👨‍🔬
**ระบบแจ้งเตือนเฉพาะทีม RD แบบมีบริบทและ actionable**

**สถานะ:** 🆕 **NEW** - รวมจาก Smart Environmental Alert + IoT Device Health Monitor

#### 🎯 **วัตถุประสงค์หลัก**
สร้างระบบแจ้งเตือนเฉพาะทีม RD ที่มีบริบทชัดเจน actionable และสื่อสารด้วยภาษาที่เข้าใจง่าย แทนที่จะเป็นแค่ตัวเลขที่ผิดปกติ

#### 💡 **ปัญหาที่พบ (Pain Point)**
ระบบแจ้งเตือนปัจจุบันมีอยู่แล้ว แต่:
- 😴 **ทีม RD เพิกเฉย** เพราะไม่เห็นว่าเร่งด่วนแค่ไหน
- 🤔 **แจ้งเตือนไม่มีบริบท** เช่น "EC = 2.3" แต่ไม่บอกว่าควรทำอะไร
- 📊 **ข้อมูลเป็นตัวเลข** ไม่ใช่ภาษาที่นักวิจัยเข้าใจ
- ⏰ **ไม่รู้ว่าเร่งด่วนหรือไม่** ทุกอย่างดูเหมือน "แจ้งเตือน" เท่ากัน
- 🔔 **Alert fatigue** - แจ้งบ่อยเกินไป จนเพิกเฉยทั้งหมด

**ตัวอย่างแจ้งเตือนปัจจุบันที่ไม่ดี:**
```
⚠️ Alert: EC = 2.3 (Zone A)
```
→ RD คิดว่า: "แล้วไง? ควรทำอะไร? เร่งด่วนไหม?"

#### 🔧 **ฟีเจอร์หลัก**

**1. Contextual Alerts (แจ้งเตือนแบบมีบริบท)**
แทนที่:
```
❌ EC = 2.3 (ไม่มีบริบท)
```
เป็น:
```
✅ 🟡 WARNING - EC สูงเกินไปในการทดลอง Oak Lettuce
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📍 Batch #2025-015 (Oak Lettuce, สูตร A)
📊 EC: 2.3 → เป้าหมาย: 1.6-1.8 (สูงกว่า +28%)
⏰ สถานะ: วันที่ 7/30 (ช่วงโต)

⚠️ ผลกระทบ:
- ใบอาจไหม้ (tip burn) ภายใน 24-48 ชม
- ผลผลิตลด 10-15%
- ทดลองนี้อาจไม่ได้ผลตามที่คาด

✅ แนะนำ:
1. ลด EC ลงเป็น 1.6-1.8 ภายใน 2 ชม
2. เช็คว่าใส่ปุ๋ยเกินหรือเปล่า
3. ตรวจสอบใบอีกครั้งใน 12 ชม

🔬 Context:
- สูตร A ควรใช้ EC 1.6-1.8
- Oak Lettuce ไวต่อ high EC
- ทดลองนี้อยู่ในช่วง Week 2 (ช่วงโต)

📞 ติดต่อ: @นักวิจัยผู้รับผิดชอบ
```

**2. Impact-Based Prioritization**
ระบุความเร่งด่วนตาม impact:

```
🔴 P0 - CRITICAL (ต้องแก้ภายใน 30 นาที)
→ PPFD = 0 (ไม่มีแสง)
→ Temp > 30°C (พืชตายได้)
→ Pump failure (พืชขาดน้ำ)

🟡 P1 - WARNING (ต้องแก้ภายใน 2 ชม)
→ EC สูง/ต่ำกว่า target 20%+
→ pH นอกช่วง
→ CO₂ ต่ำกว่า target

🟢 P2 - INFO (ตรวจสอบภายใน 1 วัน)
→ Minor fluctuations
→ ต้องเติมน้ำเร็วๆ นี้
```

**3. Experiment-Aware Alerts**
รู้ว่าแต่ละ batch เป็นการทดลองอะไร:
- **เชื่อมกับ Digital Logbook** (หัวข้อ 3)
- รู้ว่าพืชอะไร (Oak Lettuce, Butterhead)
- รู้ว่าใช้สูตรอะไร (สูตร A, B, C)
- รู้ว่าอยู่วันที่เท่าไร (Week 1, Week 2)
- รู้ว่าควรปรับอะไร

**4. Actionable Recommendations**
แนะนำการแก้ไขเฉพาะเจาะจง:
```
✅ สิ่งที่ต้องทำ:
1. ลด EC จาก 2.3 → 1.7 (ปรับลง 0.6)
   → เติมน้ำ ~30 ลิตร
2. ตรวจใบภายใน 12 ชม
3. ถ้าใบไหม้ → ถ่ายรูปส่งให้ทีม

📋 Checklist:
[ ] ลด EC ลงเป็น 1.7
[ ] เติมน้ำแล้ว
[ ] ตรวจสอบใบแล้ว (12 ชม)
[ ] แก้ไขเสร็จแล้ว
```

**5. Snooze & Acknowledge**
- **Acknowledge**: ยืนยันว่าเห็น alert แล้ว
- **Snooze**: เลื่อนเตือนไปก่อน (ถ้าไม่เร่งด่วน)
- **Resolve**: แก้ไขเสร็จแล้ว
- **Dashboard**: ดู alert ที่ยังไม่จัดการ

**6. RD-Specific Language**
ใช้ภาษาที่นักวิจัยเข้าใจ ไม่ใช่แค่ตัวเลข:
- "ใบอาจไหม้" แทนที่ "EC out of range"
- "ผลผลิตลด 10-15%" แทนที่ "High EC detected"
- "Oak Lettuce ไวต่อ high EC" แทนที่ "EC = 2.3"

#### 📊 **ตัวอย่าง Alert Types สำหรับ RD**

**1. Nutrition-Related Alerts**
```
🟡 EC ผิดปกติ → บอก impact ต่อการทดลอง
🟡 pH นอกช่วง → แนะนำวิธีปรับ
🟢 ต้องเปลี่ยนน้ำเร็วๆ นี้ → เตือนล่วงหน้า
```

**2. Growth-Related Alerts**
```
🟡 พืชเติบโตช้ากว่าปกติ → เปรียบเทียบกับ batch อื่น
🟡 ใบเริ่มเหลือง → อาจเป็น EC, pH, หรือ PPFD
```

**3. Experiment-Specific Alerts**
```
🟢 ถึงวันเก็บเกี่ยว (ตาม timeline)
🟢 ต้องถ่ายรูปพืช (สำหรับ tracking)
🟢 ต้อง measure leaf size วันนี้
```

**4. Comparison Alerts**
```
🟡 Batch A เติบโตช้ากว่า Batch B (ควรตรวจสอบ)
🟢 Batch C perform ดีกว่าปกติ (อาจเป็น best practice)
```
